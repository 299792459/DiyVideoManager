# 标签批量交换格式（给大模型阅读）

## 文件格式

- 使用 **JSON Lines（JSONL）**：**每一行一个 JSON 对象**，行与行之间用换行分隔，**不要**输出多行一个 JSON，**不要**在最外层包数组。
- 编码：**UTF-8**（无 BOM）。

## 每行字段（键名请保持不变，便于程序解析）

| 键 | 含义 | 是否可改 |
|----|------|----------|
| `i` | 视频在系统内的**整数 ID**，主键，**禁止修改** | 否 |
| `n` | 文件名（便于人读） | 可不返回；若返回应与库内一致 |
| `p` | 视频完整路径 | 可不返回；若返回用于校验，**一般不要改** |
| `t` | 当前标签名列表（字符串数组） | **是**：请把整理后的新标签写在这里 |

**最短可导入行**（大模型回传时推荐保留 `i` 与 `t`）：

```json
{"i":42,"t":["剧情","室内"]}
```

- `t` 为空数组 `[]` 表示**清空该视频的所有标签**。
- 标签名请去首尾空格；不要重复；单标签长度建议不超过 128 字符。

## 工作流程

1. 用户从本系统**导出** `tags_export.jsonl`，交给你（大模型）。
2. 你阅读每一行的 `n`、`p`（可选）与 `t`，按用户给的自然语言规则**重写 `t`**。
3. 输出**仍为 JSONL**，每行保留 **`i`**，用新标签替换 **`t`**。不要丢行、不要改 `i`。
4. 用户将新文件**导入**本系统，程序按 `i` 更新标签。

## 注意事项

- **行数**：导入行数可与导出不完全一致；仅处理文件中出现的 `i`。未出现的视频标签不变。
- **路径**：若某行带 `p` 且与库内路径不一致，导入时默认仍按 `i` 更新。若在界面勾选 **严格校验路径**，则请求 `POST /api/tags/import?strict_path=1`，此时 `p` 与库内不一致的行会被跳过。
- **不要**在文件里写注释、Markdown 或 BOM；**不要**用 CSV/XML。

## 示例

导出片段：

```json
{"i":1,"n":"a.mp4","p":"D:/v/a.mp4","t":["旧标签"]}
{"i":2,"n":"b.mp4","p":"D:/v/b.mp4","t":[]}
```

你整理后回传：

```json
{"i":1,"t":["电影","户外"]}
{"i":2,"t":["短片"]}
```
